== Overview

The Couchbase Power BI Connector integrates Microsoft Power BI with the following Couchbase data sources:

* Enterprise Analytics 
* Capella Analytics
* Couchbase Server (Analytics Service)
* Capella Operational (Analytics Service)

The connector uses the Couchbase ODBC driver to connect to these data sources.
You must first <<configure-odbc, create an ODBC Data Source Name (DSN)>> for each data source you want to use in Power BI.
The DSN defines the connection details such as the address, authentication method, and SSL settings.
After you configure the DSN, use the Power BI connector to import data from your data source into Power BI or query it using DirectQuery.

== Prerequisites

Before you begin, make sure you have the following:

[cols="1a,2a", options="header"]
|===
| Component | Requirement
| Enterprise Analytics
| * For on-premise deployments: A compatible server infrastructure. 
* For self-managed deployments: An account with your cloud provider. 
| Capella Analytics
| * An existing Couchbase Capella account.
| Couchbase Server (Analytics Service)
| * Couchbase Server 7.2.4 or later.
* The Analytics Service must be running on the target node.
| Capella Operational (Analytics Service)
| * An existing Couchbase Capella account.
* The Analytics Service must be running on the target node. 
| Power BI 
| * Tabular analytics views of your JSON documents. 
For more information, see <<connect-add-data,Connect and Add Data to Power BI>>.
|===

== Install Power BI Desktop

Before you start, install a supported version of Microsoft Power BI Desktop.
The Couchbase Power BI Connector requires the Power BI Desktop version March 2017 or later.

If necessary, download Power BI Desktop from the https://www.microsoft.com/en-US/download/details.aspx?id=58494[Microsoft downloads] page. 

NOTE: Power BI is only available for the Microsoft Windows platform;
for other platforms, consider our xref:tableau-connector::index.adoc[Tableau connector].

== Install OpenSSL

OpenSSL is a required dependency for the Couchbase ODBC driver. 

To install OpenSSL: 

. Download OpenSSL 3.x for x64 (recommended version: OpenSSL 3.5.x Light -- the current LTS version)
 from the link:https://slproweb.com/products/Win32OpenSSL.html[Shining Light Productions site].

. Install OpenSSL from the downloaded file.

. Verify that Windows has installed SSL and registered the path of the binary.
Open the command prompt (`cmd`), and enter the following:
+
[source, console]
----
openssl version
----

. If the output indicates OpenSSL is not installed, rather than returning the version that you installed, you need to add the OpenSSL binary's location to the `PATH` environment:

.. Check `C:\Program Files` or use `Find` to locate the SSL binary -- usually the path will be something like:
+
[source, console]
----
C:\Program Files\OpenSSL-Win64\bin
----

.. Add this path to the environmental variables list, found in menu:System variables[Path].

.. Now verify that Windows has registered the path of the binary.
Once again, open the command prompt (`cmd`).
It must be a fresh command prompt, as the earlier one will not have loaded the newly added environmental variables, and enter the following:
+
[source, console]
----
openssl version
----

== Install the Couchbase ODBC Driver

To install the Couchbase ODBC driver:

. Download the `couchbase-odbc-1.2-win64.msi` file from https://packages.couchbase.com/releases/couchbase-odbc-driver/1.2/couchbase-odbc-1.2-win64.msi[].
For more information, see xref:release-notes.adoc[Release Notes].

. Double-click the file to open the *Couchbase ODBC Setup Wizard*.

. Install the ODBC drivers from the downloaded file.
This installation provides both ANSI and Unicode drivers.

== Install the Couchbase Power BI Connector

To install the Power BI Connector: 

. Download `couchbase-powerbi-connector-1.2.mez` from https://packages.couchbase.com/releases/couchbase-powerbi-connector/1.2/couchbase-powerbi-connector-1.2.mez[].
For more information, see xref:release-notes.adoc[Release Notes].

. Follow https://learn.microsoft.com/en-us/power-bi/connect-data/desktop-connector-extensibility#custom-connectors[Microsoft's guide] on configuring a custom connector.

== Verify the Connector Installation

To verify if the Couchbase Power BI Connector is installed correctly:

. Open *Power BI Desktop*.
. Go to menu:Get Data[].
. In the search box, enter `Couchbase Connector`.

If the connector appears in the search results, it's installed correctly.

== Get Connection Details and Certificates

Before you configure the ODBC DSN, you must obtain the required connection details and certificates for your Couchbase data source.
These details are necessary to establish a secure connection and allow Power BI to access the data in your Couchbase cluster.

NOTE: Couchbase strongly recommends that you secure your ODBC connection using SSL.
This ensures encryption of the communication between Power BI and Couchbase.

. Sign in to the Enterprise Analytics Web Console as an Administrator.

== Configure an ODBC Data Source 

To use a Couchbase data source in Power BI, you must configure an ODBC Data Source Name (DSN).

During configuration, you can also enable logging for the ODBC driver. 
The driver saves the logs to the following directory: `Documents\Power BI Desktop\Custom Connectors`.
The log file is named `couchbase-odbc`.

IMPORTANT: Make sure to use the *ODBC Data Source Administrator (64 bit)*.
The 32-bit version is incompatible with the connector and will not work correctly.

[tabs]
====

Enterprise Analytics::
+
--
. Open the *ODBC Data Source Administration* tool.
. Go to menu:User DSN[Add].
. Select the *Couchbase ODBC Driver* (either ANSI or Unicode) from the list, and then click btn:[Finish].
. Select the data source as *Enterprise Analytics*. 
. (Optional) To enable logging, select *Collect Logs*.  
. Click btn:[Next]. 
. Enter the following DSN details:
+
[horizontal]
*Name:*:: A unique name for your data source.
*Description:*:: (Optional) A description for your data source.
*Address*:: The host and port details.
Enter the IP address of the Data (KV) node in the Couchbase cluster.
+
If you use a default port, you only need to provide the host address. 
The default SSL port is `11207`, and the default non-SSL port is `11210`.
+
If you use a non-default port, use the `host:port` format (for example, `192.168.1.100:13000`).
*Database*:: The name of the database from which the data must be extracted.
*Scope:*:: (Optional) The scope from which the data must be extracted.
+
include::partial$common.adoc[tag="extra-spaces-warning"]
*SSL Mode:*:: Select `Enable` or `Disable`. 
*Authentication:*:: Select an authentication method. +
+
Based on the SSL mode and authentication method you select, the required fields differ.
+
[cols="1a,1a,2a,3a", options="header"]
|===
| SSL Mode | Authentication | Fields | Description

| `Disable`
| `Basic`
| Username, Password
| Enter user credentials. 

| `Enable`
| `Basic`
| Username, Password, Cluster Cert Path
| Enter user credentials and the path to the root certificate you downloaded from <<ea-connection-details,Get Connection Details>>.

| `Disable`
| `LDAP`
| Username, Password

| `Enable`

For more information, see xref:enterprise-analytics:manage:manage-security/configure-ldap.adoc[Configure LDAP].

| `Enable`
| `Client Certificate`

For more information, see xref:enterprise-analytics:manage:manage-security/configure-client-certificates.adoc[Configure Client Certificates].

| `Enable`
| `Client Certificate (Encrypted Key)`

[[connect-add-data]]
== Connect and Add Data to Power BI

Business information tools rely on data organized into relational databases.
To use the Power BI connector with Couchbase Analytics, you must create tabular analytics views (TAVs) on top of your collections. 

IMPORTANT: Couchbase Analytics provides both views and tabular views. 
Only tabular views work with Power BI.

After you prepare tabular analytics views and define DSNs, use the Couchbase Power BI Connector to load data into Power BI. 

[TIP]
--
To prepare tabular analytics views, see:

* xref:enterprise-analytics:sqlpp:5a_views.adoc#TAV[Tabular Analytics Views] for Enterprise Analytics
* xref:analytics:sqlpp:5a_views.adoc#TAV[Tabular Analytics Views] for Capella Analytics
* xref:server:analytics:5a_views.adoc#tabular-analytics-views[Tabular Analytics Views] for Couchbase Server or Capella Operational
--

=== Add Data from Your Data Source

To add data from Couchbase data sources to Power BI: 

. In Power BI Desktop, select menu:Get Data[].
. Select *Couchbase Connector*.
. In the *dsn* field, enter the name of the DSN you created. 
. Select *Import* or *DirectQuery* as the *Data Connectivity mode* and click btn:[OK]. 
. Select an authentication method based on your ODBC DSN configuration. 
** `Basic` - If your DSN uses `Basic` or `LDAP` authentication, or if you're connecting to Capella Analytics or Capella Operational.
Enter your username and password.
** `Basic` (passphrase only) - If your DSN uses `Client Certificate (Encrypted Key)` authentication.
Enter your private key passphrase and leave the username field empty.
** `Anonymous` - If your DSN uses `Client Certificate` authentication. 
No certificate details required as this option provides implicit authentication.
** `Windows` - If you want to authenticate with your Windows credentials.
. Click btn:[Connect].

After connecting, Power BI displays a list of the tabular views in the database specified by your DSN. 
Select a view from the list and click btn:[Load] to import it. 
You can then use various Power BI options to visualize the data.

== Troubleshooting 

=== Performance Issues

If you notice performance issues, make sure ODBC tracing is disabled.
Enable tracing only when you need to collect logs, and turn it off afterward. 
For more information, see https://learn.microsoft.com/en-us/sql/odbc/admin/setting-tracing-options?view=sql-server-ver17[Setting Tracing Options].

=== Error Code 126: ODBC Driver Could Not Be Loaded

When creating the ODBC DSN or loading the ODBC driver, you might encounter the following error:

image::ODBC-driver-error.png[]

If this happens, try the following steps: 

. Check OpenSSL installation
* Run `openssl version` in the command prompt. 
* Make sure the version matches the driver requirements. 
* For the correct OpenSSL version, see <<install-openssl,Install OpenSSL>>.

. Verify driver installation
* Open the *ODBC Data Source Administrator (64-bit)* tool.
* Verify if the Couchbase ODBC driver is listed. 
* Reinstall if missing. 

. Check for missing dependencies
* Download the https://github.com/lucasg/Dependencies[Dependencies] tool from GitHub. 
* Open the tool and load `couchbaseodbcw.dll` and `couchbaseodbc.dll`.
* Look for any entries marked as missing in the dependency tree.

. Make OpenSSL DLLs discoverable
+
If all the previous steps are correct, you can try copying the OpenSSL DLL files directly into the Couchbase ODBC driver folder.
Sometimes Power BI cannot locate the OpenSSL dependencies at runtime, even if they're installed. 
+
* Go to `C:\Software\OpenSSL-Win64\bin\`.
* Copy these files: 
** `libssl-3-x64.dll`
** `libcrypto-3-x64.dll`
* Paste them into the Couchbase ODBC driver folder `C:\Software\couchbase-odbc\bin\`. 
* Restart and retry
** Close Power BI Desktop completely.
** Clear the Power BI cache if possible. 
** Re-open Power BI and connect again.
