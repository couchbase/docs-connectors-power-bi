== Overview

The Couchbase Power BI Connector integrates Microsoft Power BI with the following Couchbase data sources:

* Couchbase Server
* Capella Operational
* Capella Analytics
* Enterprise Analytics

The connector uses the Couchbase ODBC driver to connect to these data sources.

To use the connector, you must first create an ODBC Data Source Name (DSN) for each data source you want to use in Power BI.
The DSN defines the connection details such as the host, port, authentication method, and SSL settings.
For Couchbase Server and Capella Operational, the ODBC driver connects to the xref:server:analytics:1_intro.adoc[Couchbase Analytics Service], whereas for Capella Analytics and Enterprise Analytics, the driver connects directly to their analytical databases.

After you configure the DSN, you can use the Power BI connector to import data from your Couchbase data source into Power BI.

== Prerequisites

* Ensure that the Analytics Service is running on the target node when using Couchbase Server or Capella Operational as a data source.
* To connect to Couchbase Server (Analytics Service), use Couchbase Server version 7.2.4 or later.

== Install Power BI

Before you start, ensure you have a supported version of Microsoft Power BI Desktop installed.
The Couchbase Power BI Connector requires the Power BI Desktop version March 2017 or later.

If necessary, download Power BI Desktop from the https://www.microsoft.com/en-US/download/details.aspx?id=58494[Microsoft downloads] page. 

NOTE: Power BI is only available for the Microsoft Windows platform;
for other platforms, consider our xref:tableau-connector::index.adoc[Tableau connector].

== Install OpenSSL

OpenSSL is a required dependency for the Couchbase ODBC driver. 

To install OpenSSL: 

. Download OpenSSL 3.x for x64 (recommended version: OpenSSL 3.5.x Light -- the current LTS version)
 from the link:https://slproweb.com/products/Win32OpenSSL.html[Shining Light Productions site].

. Install OpenSSL from the downloaded file.

. Verify that Windows has installed SSL and registered the path of the binary.
Open the command prompt (`cmd`), and enter the following:
+
[source, console]
----
openssl version
----

. If the output indicates OpenSSL is not installed, rather than returning the version that you installed, then you need to add the OpenSSL binary's location to the `PATH` environment:

.. Check the `C:\Program Files` or use `Find` to locate the SSL binary -- usually the path will be something like:
+
[source, console]
----
C:\Program Files\OpenSSL-Win64\bin
----

.. Add this path to the environmental variables list, found in menu:System variables[Path].

.. Now verify that Windows has registered the path of the binary.
Once again, open the command prompt (`cmd`) -- it must be a fresh command prompt, as the earlier one will not have loaded the newly added environmental variables -- and enter the following:
+
[source, console]
----
openssl version
----

== Install the Couchbase ODBC Driver

To install the Couchbase ODBC driver:

. Download the `couchbase-odbc-1.2-win64.msi` file from https://packages.couchbase.com/releases/couchbase-odbc-driver/1.2/couchbase-odbc-1.2-win64.msi[] -- see the xref:release-notes.adoc[release notes page] for more details.

. Double-click the file to open the *Couchbase ODBC Setup Wizard*.

. Install the ODBC drivers from the downloaded file.
This installation provides both ANSI and Unicode drivers.

IMPORTANT: Make sure that you have installed the 64-bit versions of the ODBC drivers.


== Install the Couchbase Power BI Connector

To install the Power BI Connector: 

. Download `couchbase-powerbi-connector-1.0.mez` from https://packages.couchbase.com/releases/couchbase-powerbi-connector/1.0/couchbase-powerbi-connector-1.0.mez[] --
see the xref:release-notes.adoc[release notes page] for more details.

. Follow https://learn.microsoft.com/en-us/power-bi/connect-data/desktop-connector-extensibility#custom-connectors[Microsoft's guide] on configuring a custom connector.

== Verify the Connector Installation

To verify if the Couchbase Power BI Connector is installed correctly:

. Open *Power BI Desktop*.
. Go to menu:Get Data[].
. In the search box, enter `Couchbase Connector`.

If the connector appears in the search results, it is installed correctly.

== Create an ODBC Data Source

To connect Power BI to your Couchbase data source, you must configure an ODBC Data Source Name (DSN)

This section explains how to set up an ODBC Data Source Name (DSN) a Couchbase data source using the Couchbase ODBC driver.

You can create an ODBC data source for each Couchbase data source you want to use in Power BI. 

You can also enable logging for the ODBC driver. The logs are stored in <location>. 

IMPORTANT: Be careful to use the *ODBC Data Source Administrator (64 bit)*; there is an *ODBC Data Source Administrator (32 bit)*
available, but this will not work with the connector.


[tabs]
====

Capella Analytics::
+
--
. Open the *ODBC Data Source Administration* tool.
. Select the *User DSN* tab and click btn:[Add].
. Select the *Couchbase ODBC Driver* (either ANSI or UNICODE) from the list, and then click btn:[Finish].
. On the next window, select btn:[Capella Analytics] as the data source. 
. (Optional) To enable logging, select btn:[Collect Logs].  
. Click btn:[Next]. 
. Enter the following DSN details:
+
[horizontal]
*Name:*:: A unique name for your data source.
*Description:*:: An optional description.
*Connection String:*:: The exact connection string copied from <<capella-columnar-connection-string,here>>.
*Database*:: The name of the database from which the data must be extracted.
*Scope:*:: The specific scope from which the data must be extracted. +
For two-part scopes, use a `/` to separate the two parts. 
For example, if the scope is `travel-sample.inventory`, enter it as `travel-sample/inventory`.
*User*:: The username created in the <<capella-columnar-user-account,User Account section>>
*Password*:: The password created in the <<capella-columnar-user-account,here>>. +
NOTE: For Power BI connections, the *Username* and *Password* fields are optional.
However, in the Power BI tool, you must provide these credentials when prompted.
. Click btn:[OK] to save the DSN and complete the setup.
--

+
[cols="1a,1a,2a,3a", options="header"]
|===
| SSL Mode | Authentication | Fields | Description

| `Disable`
| `Basic`
| * Username +
  * Password
| Your account credentials. 

| `Enable`
| `Basic`
| * Username +
  * Password +
  * Cluster Cert Path
| Account credentials and the path to your downloaded root certificate.

| `Disable`
| `LDAP`
| * Username +
  * Password 
| Your LDAP directory credentials. 

| `Enable`
| `LDAP`
| * Username +
  * Password +
  * Cluster Cert Path 
| LDAP credentials and the path to your downloaded root certificate.

| `Enable`
| `Client Certificate`
| * Client Cert Path +
  * Client Key Path +
  * Cluster Cert Path
| Paths to your client certificate, private key, and root certificate.

| `Enable`
| `Client Certificate (Encrypted Key)`
| * Client Cert Path +
  * Client Key Path +
  * Key Password +
  * Cluster Cert Path +
| Paths to your certificates and encrypted key, including the key passphrase.    
|===
*Advanced Parameters:*:: Additional parameters for the ODBC connection.
. Click btn:[OK] to save the DSN and complete the setup.

== Use the Couchbase Power BI Connector

Business information tools rely on data organized into relational databases.
To use the Power BI Connector, you must create tabular analytics views of your JSON documents. 
// For Capella columnar, see xref:query:workbench.adoc#TAV[Save Results as a Tabular View] or xref:sqlpp:5_ddl.adoc#TAV[Tabular Views].
For self-managed CBAS, see xref:server:analytics:run-query.adoc#Using_analytics_workbench[the workbench docs] or 
xref:server:analytics:5a_views.adoc[tabular views].

After you prepare tabular views and define DSNs in Power BI, you use the Couchbase Power BI Connector to load data into Power BI. 

=== Use the Connector to Add Data to Power BI

To add data from Couchbase data sources to Power BI, follow the steps in the Microsoft documentation on https://learn.microsoft.com/en-us/power-bi/connect-data/desktop-connect-to-data[Connecting to data sources in Power BI desktop].

. In Power BI Desktop, select menu:Get Data[].
. Select *Couchbase Connector*.
. In the *dsn* field, enter the name of the DSN you created. 
. Select a *Data Connectivity mode* and click btn:[OK]. 
. Select an authentication method based on your ODBC DSN configuration. 
** Basic / LDAP / Capella Analytics / Capella Operational: Select `Basic` and enter your username and password.
** Client Certificate: Select `Anonymous`. 
It provides implicit authentication using your configured certificate. 
** Client Certificate (Encrypted Key): Select `Basic` and enter your password. 
You can leave the username field empty.
+
You can also select `Windows` in Power BI to use your current Windows credentials. 

After you connect, Power BI displays a list of the tabular views from the database specified by the DSN. 
Select a view from the list and click btn:[Load] to import the view. 
Once loaded, you can use Power BI options to analyze and visualize the data.
